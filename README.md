## 프로젝트 구성

### 3차 프로젝트 - RechargeApp

1. URL : RechargeApp_3rdProject
   
2. 설명 : 전기차 충전소 위치/정보 제공 및 충전 중 미디어 컨텐츠 추천/운세/커뮤니티 기능으로 휴식시간을 제공하는 라이프스타일 플랫폼 앱
   
3. 담당 구현 기능
  - 앱(Android) 환경에 맞추어 영화/음악 추천 프론트 재설계 구축
  - 외부 API 활용 영화/음악 추천 도메인 전반 설계 및 구현
  - Python FastAPI로 AI 추천 기능을 구현하여 Spring Boot 백엔드 연동 구조 설계
  - 사용자 중심 기능(마이페이지, 즐겨찾기) 구현을 통한 개인화 경험 강화

4. 기술 스택
  - React Native
  - Spring Boot(JAVA 17)
  - Axios
  - MyBatis
  - FastAPI
  - 외부 API(TMDB, Apple Music RSS, iTunes Search API, Gpt Open AI API, WebClient)
    
5. 프로젝트를 통해 배운점
  - 외부 API(TMDB) 호출 로직이 여러 Service 계층에 분산되면서 중복 코드와 유지보수 부담이 발생할 수 있음을 경험함
    → 외부 API 연동 로직을 공통 Client 계층으로 분리해야 할 필요성 인식
  - 기능 단위 구현에 집중할 경우 응집도가 낮아질 수 있음을 확인
    → API 호출 책임 분리와 계층 역할 정의의 중요성 학습
  - 음악 도메인에서 Spotify API 도입을 검토하며 인증 구조와 운영 복잡도가 구현 난이도에 큰 영향을 준다는 점을 경험
    → 프로젝트 일정·팀 역량을 고려해 인기 곡 중심 구조로 기능 범위 조정
  - 외부 API 도입 시 기술적 가능성뿐 아니라 인증 방식, 유지보수 비용, 확장성까지 사전 검토가 필요하다는 점을 학습
  - 향후에는 API 연동 구조 사전 검증과 단계적 확장이 가능한 설계를 적용할 계획
